DQ - Dashboard

Intoduction#

Dashboards are primary window through which user gain insights into the data in a user interface. A good dashboards should be created with a balance between business and technical insights. Business teams strive for good quality of data to design strong marketing and growth strategies. If the quality of the data is impacted business team being up to date is of at most important. On other side, with volume of data increasing day by day monitoring becomes tedious task for technical team. Technical dashboards alters technical team if data quality is been impacted. Identifying issue at early stage helps technical team with good time to fix issues and eliminates rework/ cleanup on bad data. Wavicle data quality bot features with business and technical dashboard to be leveraged by an organization in one single platform. Dashboards are per-defined reducing the effort required to creating one. On successful run, dashboards are refresh with latest data from DQ bot stats tables. Dash boards also comes with export option helping users share information across organization in matter of minutes.

BUSINESS DASHBOARDS#

Business charts are at pillar level to help business team. Attributes are mapped into 6 pillars based on their behavior.

Pillar's and attributes are listed below:

Uniqueness: Measures quality of data duplication based on Uniqueness percentage.

Completeness: Measures quality of missing data based on not null percentage.

Consistency: Measures quality of data through matching pattern based on regex and like pattern.

Accuracy: Measure quality of data with in range based on range percentage set up user.

Conformity: Measure quality of data with valid date format based date format percentage.

Integrity: Measure quality of trust worthiness in the data based on nUll, regex, like percentage.

Last run date Charts(table level)#

Data compliance: Based on user selection attributes are categorized into good data or bad data. Good data are charted in compliant meter and vice versa.

Counts of Data check: Count of selected data checks in a bar chart.

Data Check Snapshot: Displays percentage for data checks selected.

Data Check Snapshot#

Table, column and frequency drop down areas allows user to slice or dice data based on their interest.

Last Run Stats: Statistical metrics like Mean, Median, Mode, Standard deviation are displayed.

Good to Bad data ration: Quality check on high level. Data are grouped into good or bad based on user selection. Allows user to decide if the fix has to be limited or delayed.

Data issue affecting data quality: Bar chart developed with Pareto principle. This chart ranks issues contributing to bad data top to bottom.

Trend charts#

Table, column and frequency drop down areas allows user to slice or dice data based on their interest.

Completeness: Break downs each pillar by 12 months.

Performance metrics of pillars: Compares upward or downward trend between months selected.

Trend In Range: Charts counts of records fall with range selected by the user based on frequency selected.

Rows Scanned: Total count of records for selected time range.

Outside Range: Summaries total count of records falling out side the range selected by user.

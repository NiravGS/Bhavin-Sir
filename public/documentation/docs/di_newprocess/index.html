DI - Create New Process

Building an ELT/ETL or a data migration process has never been so easy. This cloud based, metadata driven approach makes the process very simple and with minimal information and with no coding your process would be live in a matter of minutes.

Step 1 - Login#

  • Login into the web app with your credentials.
  • Open the Data Integration suite
  • Click on the Create New Process

Step 2 - Source Information#

An input wizard opens up as shown below,

Process Name - Provide name which will help you identify the functionality of the process. Process names cannot have spaces between them. Replace spaces with - or _.

Description - A more detailed description of what your process is intended for. This field is optional, but it is a good practice to provide a description.

Cloud Type - Select your cloud environment. By default, it is AWS.

Cloud Bucket - Provide the datalake bucket name here. Just the bucket name would suffice. Example: input "test-aws-datalake-bucket" instead of "s3://test-aws-datalake-bucket".

Source Type - Choose your source type. Is your your source system a Database or a Filesystem. In this example we will take database as source. Check here for filesystem as source.

Note: On selection of the source type, you may notice that there are few input fields popping up. We will get to that in a bit.

Source System - Select your source system (connection) from the dropdown. These are systems that you or your team has already added. If you are building a process for a new source system, click the link Not able to find the source? Click here to add a new source.

You will now be able to input your connection details of the new system and upon completion, you will be redirected to the same page where you left.

Source Database - Once the source is is selected/added, select your source database.

Your window should now look something as below,

Source Tables - All the tables present inside the database chosen are displayed as a dropdown here. You will be able to select multiple tables by selecting from dropdown. You also have a option to select all the tables, just click the link select all.

Now that we have provided all the source information, it's time to move to target information. Simply click on Next to advance to target information page.

Step 3 - Target Information#

You should now be able to see the target information input wizard which should look like below,

Target Type - Same as source information, select the target type from the drop down. It can either be Filesystem or Database. For this example, we will stick to Database as target.

Target System - Select your target system from the systems that are already added. Upon selection of target system, a few more input fields should pop up.

Target Database - Select the target database where the data needs to be loaded.

Raw Schema - Select the raw schema

Stage Schema - Select your stage schema

Target Schema - Select your final target schema where the transformed data will be loaded.

Note: To know more about usage Raw Schema and Stage Schema, Click here.

On selection of the respective schema's you will see another set of input fields pop up as below,

Create Target Table - This check box is checked by default. For any new process created we assume that there is no corresponding target table present. If checked, The process would automatically create the tables in target with the same name as source table.

Source Table - These are the tables selected as part of the process in the source information section. This field cannot be edited here. I you wish to add or remove a table, you will have to go back using the Back button.

Target Table - This field is disabled by default. For any new process created we assume that there is no corresponding target table present. The process would automatically create the tables in target with the same name as source table. If the target tables are already present, uncheck the Create Target Table check box, so you will be able to map a source table to the already created target tables within the selected target schema.

Ingestion Type - This field allows you to specify the type of ingestion you would like to perform at the individual table level. The supported ingestion types are Full, Incremental and Master SCD. To know more about these ingestion types Click here.

CDC Columns - This field is disabled by default and it is enabled if the ingestion type is 'Incremental' or 'Master SCD'. This field captures the columns from the source table that are required to perform the change data capture between executions. Click here to know more on how CDC is performed.

SCD Type - This field allows you to select the type of SCD you would like to perform. Currently we support, Type 1 and type 2 SCD.

SCD Columns - Similar to CDC columns, we would capture the field required for performing the SCD. Click here to know more on how SCD is performed.

Now that all the target information are provided, let's move the the next page.

Step 4 - Schedule and Review#

Now you are in the scheduler page. You can add as many schedule as you wish here for the process. The schedules can be one of the below,

  • One time
  • Daily
  • Weekly
  • Monthly

Choose the starting date, time of execution and the frequency as per the requirements and click on Add Schedule . You should see the schedule added in the right side as shown in below screenshot,

After adding the schedule, proceed to the review page. Here you will see all the information you provided for the process.

Validate all the information such as the source and target tables, SCD type, schedule and once validated, click on the Create Process. Your pipeline is now live. The process will automatically trigger with the help of a CRON in the backend. You will also have the ability to run the jobs manually from the UI.

On successful creation of the process, you will get pop up message and will be redirected to the View Process page.

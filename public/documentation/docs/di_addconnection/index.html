DI - System Configurations

To build your data ingestion pipelines you will need to have the system credentials that you are connecting to. This feature in the tool allows you to add the system connection credentials to be saved in a secure metadata database which can then be used while creating your ingestion pipelines.

How to add a new system ?#

Adding your source and target systems is a one time activity unless there is a credentials change made to connect to a system. You can add any number systems that you would like to connect to in a very few steps as outlined below,

Step 1

In the Data Integration window, click the Create Connection located on the left side bar.

Step 2

In the input wizard, provide the required information.

Connection Name : A unique connection name that helps you distinguish between your systems.

Source Type : Select the type of system. In this case, whether it is the Filesystem or a Database Note: By default, it is set to Filesystem.

Source Database Type/ Source Filesystem Type : Based on your selection in the above field you will be shown various option here for individual types. Select your corresponding input here.

- Source Type Database
- MySQL
- Aurora (MySQL)
- Redshift
- Snowflake
- SQL Server
- Postgres
- Oracle
- Source Type Filesystem
- AWS S3
- HDFS
- Local

Step 3

Upon providing the required input above, you will be shown a modal where you will have to provide the connection parameters for the selected system.

The required fields in the modal would differ based on the inputs provided in the previous step. For this example, we chose the type as Database and Database as MySQL. Below are the required fields for connecting to MySQL,

  • HOST
  • USERNAME
  • PASSWORD
  • PORT
  • DATABASE NAME
  • Additional Settings (Optional)

To know more about other systems and their required fields,

After providing the required information, Test the connection. Upon a successful test you will be able to Save the connection.

You have now successfully linked a new system.

How to edit details of existing system ?#

  1. In order to edit the system connection settings, In the Data Integration window, click the View Connections located on the left side bar.

  2. This will show you the list of connections that are already added.

  3. Click on the connection for which you wish to update the information. This is show provide an option to Edit or Delete connection. Click on Edit .

  4. This will open a modal will values pre-populated except the password.

  5. Any information updated here, including the password, will eventually be updated in the metadata. Be sure to test the connection before saving.

How to delete details of existing system ?#

  1. In order to delete the system, In the Data Integration window, click the View Connections located on the left side bar.

  2. This will show you the list of connections that are already added.

  3. Click on the connection which you wish to delete. This is show provide an option to Edit or Delete connection. Click on Delete .

  4. Confirm your action in the pop up and your system will be removed from metadata.

Note: Deleting a system, will NOT delete the processes associated with the system. BUT, the processes would eventually fail as there is no such system exists in metadata.

DQ/DI - Deployment

Code deployment#

1) Spin up compute layer preferably EC2 instance.

2) Install Python 3 and Apache Spark for file based sources

3) Download DQ BOT code in data quality repository from Git hub

4) Refer to requirement.txt for all python packages to be installed.

5) Run following command to install all packages: pip install -r requirements.tx

6) Run pip list command to verify all packages are installed.

7) Set up corn job to trigger run_dq_bot.sh script based on run schedule created in UI.

8) Once DQBOT Database set up is complete edit dq_config.yaml and db_config.cnf files to set up DB's connection details.

DQ Database deployment#

1) Spin up RDS instance preferably MYSQL.

2) Create a DQBOT database. Please refer DQBOT data model for more details.

FAQ'S#

1) Is process specific to account ?

Ans: Yes, dashboards can be shared across organization but only owner of the process can edit it

2) Is there limit for number of process ?

Ans: No

3) Does dashboard gets refreshed real time ?

Ans: No, dashboards are refresh after scheduled run

4) Data quality checks performed on entire database or on delta's ?

Ans: Every run scans entire data set and dash boards reflect data quality at specific time

5) What happens when scheduled run fails ?

Ans: Users are alerted through registered emails. Please refer alerts guide for more reference.

6) Most common reason for scheduled run failure or dashboard issue ?

Ans:

1) Regex or like pattern not properly defined.

2) Check for RDS connection.

3) Scheduled run is yet to be complete

4) computer layer down

7) What are file format supported ?

Ans: CSV, PSV, Textfile, JSON, AVRO

8) Can custom dashboards build ?

Ans: No dashboards are per-build. Please reach out to technical team for custom build DQ Checks.

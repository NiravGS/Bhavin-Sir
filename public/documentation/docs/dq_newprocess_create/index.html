DQ - New Process

NEW PROCESS#

On successful creation of source connection select the process to proceed to set up DQ rules.

Set up new process by selecting:#

Database type: Select database / data warehouse on which process needs to be created.

Supported db's:
-> Redshift
-> MySQL
-> Aurora
-> Post gres
-> SQL- Server
-> Oracel
-> Snow flake

Source Name: Describes name of the system, release or a product which host data set to be monitored for data quality.

Process Name: A unique name to define, track & schedule DQ job flow. Along to Visualize and share dashboards. e.g.: database name

FETCH OBJECTS#

Collection of schema object from the source system:

Fetch: Allow the DQ bot to scan source database to get the schema objects on which data checks can be mapped.

Cancel: Return to Connection screen.

Create configuration: On successful schema scan DQ bot allows user to create configuration.

CREATE CONFIGURATIONS(MAKE SELECTION)#

Selecting table and mapping columns on the table with data check attributes.

Table section: List of all tables in the source selected.

Column section: On selecting a table all columns related to the table are listed.

Data check section: List of all data check attributes DQ bot features with.

Save & Continue: Selections are saved to the DQ database for scheduled run.

Finish: On complete click finish to proceed to review screen.

User selects table in which data quality check to be performed. Then selects columns and maps it to the attributes to which data check needs to be performed. Single column can be mapped to multiple data checks. Like wise more than one table and more than one column can be mapped to data checks to construct a process.

PROCESS REVIEW#

List of all tables, columns and data checks selected as part of the process.

Update configuration: Takes back to the create configuration screen for updates.

Back: Returns to the home screen.

Manage: List more options in DQBOT.

ALERTS#

CREATE SCHEDULE#

Core part of the automation is to run the process at schedule time with out manual efforts. Create schedule feature allows the user to create a run schedule on which each process will be execute in the compute layer. Create the schedule by selecting date, time and interval.
